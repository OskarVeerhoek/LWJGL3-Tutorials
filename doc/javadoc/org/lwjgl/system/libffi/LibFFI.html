org.lwjgl.system.libffi

Class LibFFI

  • java.lang.Object
    • org.lwjgl.system.libffi.LibFFI

  • public final class LibFFI
extends java.lang.Object
    Native bindings to the libffi library.

    The foreign function interface provides a mechanism by which a function can generate a call to another function at runtime without requiring knowledge of the called function's interface at compile time. This enables use of native libraries that LWJGL does not provide bindings for.

    libffi assumes that you have a pointer to the function you wish to call and that you know the number and types of arguments to pass it, as well as the return type of the function.

    The first thing you must do is create an FFICIF object that matches the signature of the function you wish to call. This is a separate step because it is common to make multiple calls using a single FFICIF. The cif in ffi_cif stands for Call InterFace. To prepare a call interface object, use the function prep_cif. To call a function using an initialized ffi_cif, use the call function.

    LWJGL note: The closure and raw APIs are not exposed.

      • ffi_prep_cif

        public static int ffi_prep_cif(java.nio.ByteBuffer cif,
               int abi,
               int nargs,
               long rtype,
               java.nio.ByteBuffer atypes)
        Prepares an FFICIF structure for use with call.
        Parameters:
        cif - the FFICIF structure to prepare
        abi - the calling convention to use. One of:
        SYSV, STDCALL, THISCALL, FASTCALL, MS_CDECL, WIN64, UNIX64, DEFAULT_ABI
        nargs - the number of arguments
        rtype - points to an FFIType that describes the data type, size and alignment of the return value
        atypes - an array of nargs pointers to FFIType structs that describe the data type, size and alignment of each argument
        Returns:
        Upon successful completion, ffi_prep_cif returns OK. It will return BAD_TYPEDEF if cif is NULL or atypes or rtype is malformed. If abi does not refer to a valid ABI, BAD_ABI will be returned.

      • ffi_call

        public static void ffi_call(java.nio.ByteBuffer cif,
            long fn,
            java.nio.ByteBuffer rvalue,
            java.nio.ByteBuffer avalue)
        Provides a simple mechanism for invoking a function without requiring knowledge of the function's interface at compile time.
        Parameters:
        cif - a FFICIF structure. It must be initialized with prep_cif or prep_cif_var before it is used with ffi_call.
        fn - the function to call
        rvalue - a pointer to storage in which to place the returned value. The storage must be sizeof(ffi_arg) or larger for non-floating point types. For smaller-sized return value types, the ffi_arg or ffi_sarg integral type must be used to hold the return value.
        avalue - an array of pointers from which the argument values are retrieved
