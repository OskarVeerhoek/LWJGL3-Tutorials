org.lwjgl.openal

Class SOFTBufferSamples

  • java.lang.Object
    • org.lwjgl.openal.SOFTBufferSamples

  • public final class SOFTBufferSamples
extends java.lang.Object
    Native bindings to the SOFT_buffer_samples extension.

    This extension provides a more flexible mechanism for loading buffer data, as well as a method to retrieve buffer data. Unextended OpenAL only provides a method to specify a single buffer format when loading data, which defines the data given by the application. The AL is given leeway in converting the data, so that it is possible or more efficient to use internally. However, there are some drawbacks to this approach:

    • The conversion done by the implementation is hidden from the app. This makes it difficult for the app to know what kind of precision it will have, and impossible to request a storage precision.
    • Conversion is not guaranteed, so the application can be restricted in the formats that can be loaded depending on the implementation.
    If the application could specify the internal storage format, as well as use a separate format to specify the incoming data's format, it would allow to add more input formats (signed 8-bit, 32-bit int, and float, for example), with no undue burden placed on the implementation beyond needing some conversion routines. The application can then be assured that many different formats can be loaded, even if storage is restricted to a comparatively small subset.

    In addition, unextended OpenAL does not have any methods for updating only a portion of a buffer, nor a method to retrieve the data from a buffer.

      • alBufferSamplesSOFT

        public static void alBufferSamplesSOFT(int buffer,
                       int samplerate,
                       int internalformat,
                       int samples,
                       int channels,
                       int type,
                       java.nio.ByteBuffer data)
        Removes a buffer's sample data and replaces it with new data. The named buffer must be a buffer ID returned by GenBuffers, and not currently be attached to or queued on a source. The given samplerate is the number of sample frames per second the data will play at, and the internalformat is the storage format used. The number of uncompressed sample frames to load is specified by samples.

        The channels and type parameters specify the channel configuration and sample type of the provided data. The samples will be converted to the internal storage format as needed, however the channel configuration must match the channel configuration of the storage format or an INVALID_ENUM error will be generated.

        The data pointer should hold enough data as specified by the channels, type, and samples parameters, and will be copied and converted into the buffer's storage. It may also be NULL, in which case the stored data will be undefined.

        If AL_SOFT_loop_points is supported, a successful call will reset the start and end loop points to 0 and samples respectively.

        Parameters:
        buffer - the buffer ID
        samplerate - the number of sample frames per second the data will play at
        internalformat - the internal format. One of:
        MONO8_SOFT, MONO16_SOFT, MONO32F_SOFT, STEREO8_SOFT, STEREO16_SOFT, STEREO32F_SOFT, QUAD8_SOFT, QUAD16_SOFT, QUAD32F_SOFT, REAR8_SOFT, REAR16_SOFT, REAR32F_SOFT, 5POINT1_8_SOFT, 5POINT1_16_SOFT, 5POINT1_32F_SOFT, 6POINT1_8_SOFT, 6POINT1_16_SOFT, 6POINT1_32F_SOFT, 7POINT1_8_SOFT, 7POINT1_16_SOFT, 7POINT1_32F_SOFT
        samples - the number of uncompressed sample frames to load
        channels - the channel configuration of the provided data. One of:
        MONO_SOFT, STEREO_SOFT, QUAD_SOFT, REAR_SOFT, 5POINT1_SOFT, 6POINT1_SOFT, 7POINT1_SOFT
        type - the type of the provided data. One of:
        BYTE_SOFT, UNSIGNED_BYTE_SOFT, SHORT_SOFT, UNSIGNED_SHORT_SOFT, INT_SOFT, UNSIGNED_INT_SOFT, FLOAT_SOFT, DOUBLE_SOFT, BYTE3_SOFT, UNSIGNED_BYTE3_SOFT
        data - the buffer data

      • alBufferSubSamplesSOFT

        public static void alBufferSubSamplesSOFT(int buffer,
                          int offset,
                          int samples,
                          int channels,
                          int type,
                          java.nio.ByteBuffer data)
        Update a buffer with new data. buffer is the ID of the buffer to modify. offset is the first sample frame in the internal storage to be modified, and samples is the number of sample frames to modify. They must not extend beyond the existing internal storage length. The channels and type parameters specify the channel configuration and sample type of the provided data buffer, specified by data, which will be copied and converted to the buffer's storage format. As with BufferSamplesSOFT, the channel configuration must match the storage format's channel configuration or an INVALID_ENUM error will be generated.

        When modifying a buffer that's playing on a source, an application must take care to not modify the section that is currently being played. The read-only source attributes BYTE_RW_OFFSETS_SOFT and SAMPLE_RW_OFFSETS_SOFT may be used to retrieve the read and write cursor offsets. Behavior is undefined if an attempt is made to modify buffer data between the read and write offsets.

        Parameters:
        buffer - the buffer ID
        offset - the first sample frame in the internal storage to be modified
        samples - the number of sample frames to modify
        channels - the channel configuration of the provided data. One of:
        MONO_SOFT, STEREO_SOFT, QUAD_SOFT, REAR_SOFT, 5POINT1_SOFT, 6POINT1_SOFT, 7POINT1_SOFT
        type - the type of the provided data. One of:
        BYTE_SOFT, UNSIGNED_BYTE_SOFT, SHORT_SOFT, UNSIGNED_SHORT_SOFT, INT_SOFT, UNSIGNED_INT_SOFT, FLOAT_SOFT, DOUBLE_SOFT, BYTE3_SOFT, UNSIGNED_BYTE3_SOFT
        data - the buffer data

      • alGetBufferSamplesSOFT

        public static void alGetBufferSamplesSOFT(int buffer,
                          int offset,
                          int samples,
                          int channels,
                          int type,
                          java.nio.ByteBuffer data)
        Reads the number of sample frames, given by samples, starting at offset, and converts them to the channel configuration and sample type specified by channels and type, before writing to the provided data buffer pointer. An INVALID_VALUE error is generated if offset and samples reach beyond the range of the buffer. The channel configuration must match the internal storage format's channel configuration, or else an INVALID_ENUM error will be generated.
        Parameters:
        buffer - the buffer to read
        offset - the sample frame offset
        samples - the number of sample frames to read
        channels - the target channel configuration. One of:
        MONO_SOFT, STEREO_SOFT, QUAD_SOFT, REAR_SOFT, 5POINT1_SOFT, 6POINT1_SOFT, 7POINT1_SOFT
        type - the target sample type. One of:
        BYTE_SOFT, UNSIGNED_BYTE_SOFT, SHORT_SOFT, UNSIGNED_SHORT_SOFT, INT_SOFT, UNSIGNED_INT_SOFT, FLOAT_SOFT, DOUBLE_SOFT, BYTE3_SOFT, UNSIGNED_BYTE3_SOFT
        data - buffer to write to
